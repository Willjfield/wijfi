I need to think about whether the image prompts serve a valuable purpose. On the one hand it could speed up the prototyping process. On the other, it can be used to thoughtlessly, and perhaps unethically, take other peoples' creative work without necessarily crediting them appropriately. 

Images aside, I'd like to build in specific accessibility guidelines like [W3's Web Accessibility Initiative](https://www.w3.org/WAI/) for the agent to reference and maybe have an interface for users with specialized use cases. (Apps will have some unique needs depending on whether the map is intended for people with visual impairment vs. motor impairment.) And I'd like to flag for the user any changes that may impede accessible design. For example, if the user asks the agent to make a font italics, it can do so but should respond that this will be difficult for certain users. Lastly, I'd like to consider a script that could automatically generate style.json copies geared towards different accessability needs. Imagine if it became common for web maps to have a dropdown that offered not just light vector, dark vector, and satellite versions but also for a range of accessibility needs.
